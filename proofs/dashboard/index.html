Proof Dashboard

Exact Proof Map for the Portability Model

This page is documentation, not a calculator. It only shows named Lean definitions, named theorem hypotheses, and exact dependency paths. Live numeric controls were removed because browser arithmetic would only yield floating-point approximations, and the old UI implied stronger claims than the proofs actually expose.

Exact Scope

Only exact definitions and exact theorem-level consequences are surfaced here. If a statement depends on hypotheses, those hypotheses are named next to the claim.

Why No Sliders

A slider would suggest measurable continuous control and a browser-side numeric answer. This file now refuses that implication. Exact reals stay symbolic in the UI.

Critical Correction

R²_target is not presented as R²_source × ρ. The exact DGP theorem gives a rational transport formula instead, and this page now uses that exact statement.

UI Honesty Rule

When the Lean files only show that one quantity feeds another, the UI says “dependency only.” It does not invent a monotone direction or a biological estimate.

Exact Definitions and Exact Formula Steps

Each card below names the Lean object directly. The page distinguishes definitions from theorems, and it calls out where the DGP block itself mixes continuous time and floored time.

Step 1. Scaled Parameters

+

bigM  = 4 * Ne * mig
Depends on

Changes next

Definition Mixed Time Semantics

Step 2. Divergence Pieces

The exact DGP block uses both real time and floor(t_div).

+ 
fstEquilibrium = 1 / (1 + theta + bigM)
hetDecayFactor = (1 - 1 / (2 * Ne)) * (1 - theta / (2 * Ne))
fstTransient   = fstEquilibrium * (1 - hetDecayFactor ^ floor(t_div))
Depends on

Ne, mu, mig, t_div.

Exact honesty note

fstTransient uses Nat.floor t_div exactly in this file. The UI now says that directly instead of implying a single continuous-time pipeline.

fstEquilibrium, PGSEvolutionaryModel.hetDecayFactor, fstTransient
Definition Model-Level Note

Step 3. Shared-Signal Factors

These are exact definitions in the DGP block.

+ 
ldRetention = exp(-2 * recomb * t_div)
mutErosion = exp(-theta * tau)
migBoost   = 1 + bigM * tau / (1 + bigM)
Depends on

recomb, t_div, theta, tau, bigM.

Exact honesty note

The formulas above are exact definitions in DGP.lean. Some surrounding comments describe them as continuous or low-parameter surrogates of deeper recurrences. This page therefore labels them as model definitions, not approximation-free biological laws.

Step 4. Transported Signal

ρ is derived from transported signal variance, not vice versa.

+

portabilityRatio = signalTransportFactor
Depends on

fstTransient, ldRetention, mutErosion, migBoost.

Changes next

targetSignalVariance, then exact deployed metrics.

Step 6. Deployed Metrics

The exact end-to-end target formulas replace the old calculator outputs.

+ 
R2_target =
  (R2_source * signalTransportFactor) /
  (1 - R2_source + R2_source * signalTransportFactor)

  Phi(sqrt((R2_source * signalTransportFactor) / (2 * (1 - R2_source))))

  prevalence * (1 - prevalence) * (1 - R2_target)

Boundaries

What the Page Refuses to Imply

Deep honesty in this file means saying where the proofs stop, where the model block introduces its own definitions, and where a browser cannot remain exact.

The browser cannot evaluate real-valued exponentials, square roots, or Phi exactly. That is why the page now shows formulas and theorem names instead of live numbers.

Exactness in this UI means exact symbolic statements that map directly to Lean names, not floating-point outputs.

Excluded

portabilityDecayRate is not surfaced as exact

The docstring in DGP.lean describes the decay-rate story as approximate. The old dashboard promoted it anyway. This page does not.

portabilityDecayRate
Boundary Label

Continuous-factor formulas are labeled as model definitions

exp(-2 * recomb * t_div) and exp(-theta * tau) are shown because they are exact definitions in the DGP block. The page does not silently upgrade them into approximation-free empirical laws.
